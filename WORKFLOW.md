# 赤金最高生产力三人组 — 求解流程

## 1. 数据来源

从 [PRTS wiki 后勤技能一览](https://prts.wiki/w/后勤技能一览) 解析制造站所有技能，生成结构化数据。

**产出文件：**
| 文件 | 说明 |
|------|------|
| `manufacturing_skills.json` | 105条技能的完整结构化数据 |
| `制造站_重新分类.md` | 按机制分类的表格，含干员和精英要求 |
| `制造站未实现技能跟踪.md` | （旧版跟踪表，以重新分类版本为准） |

**数据字段（每条技能）：**
```json
{
  "id": 29,
  "name": "金属工艺·β",
  "desc": "进驻制造站时，贵金属类配方的生产力+35%",
  "recipe": "gold",
  "operators": [{"name": "砾", "elite": 1}],
  "prod_flat": "+35%",
  "category": "纯数值",
  "subcategory": "贵金属",
  "simple": true
}
```

## 2. 技能分类体系

按实现难度递进，共 16 类：

| 序号 | 分类 | 赤金条数 | 难度 | 说明 |
|------|------|----------|------|------|
| V1 | 纯数值 | 37 | ⭐ | 无条件固定加成，已完成 |
| V2 | 渐进加成 | 5 | ⭐ | 随时间增长到上限 |
| V3 | 设施数量 | 4 | ⭐ | 按贸易站/宿舍/训练室数量 |
| V4 | 设施干员 | 2 | ⭐⭐ | 检查其他设施内的特定干员 |
| V5 | 派系+小队 | 3 | ⭐⭐ | 统计基建内派系人数 |
| V6 | COOP同站 | 1 | ⭐⭐ | 需特定干员同制造站 |
| V7 | 技能联动 | 5 | ⭐⭐⭐ | 同类技能互相加成（含配合意识） |
| V8 | 归零机制 | 5 | ⭐⭐⭐ | 清零其他干员→按电厂补偿 |
| V9 | 心情相关 | 4 | ⭐⭐ | 管理/消除/落差 |
| V10 | 仓库→产能 | 2 | ⭐⭐ | 仓库容量转生产力 |
| V11 | 中间产物 | 13 | ⭐⭐⭐⭐ | 涉及跨设施资源链 |
| V12 | 技能合并 | 1 | ⭐ | 合并技能类别 |

> **分类修正记录：** #46 配合意识从"纯数值"更正为"技能联动"——它依据其他干员提供的生产力做乘法（每5%→+5%，最多+40%）。

## 3. 求解器架构

```
solver_v1_simple.py          ← V1 纯数值求解器
solver_v2_ramp.py            ← V2 加入渐进加成 (待建)
...
solver_final.py              ← 终版：整合所有机制
```

**每个版本的求解器结构：**
1. 加载 `manufacturing_skills.json`
2. 筛选：只取 `recipe ∈ {gold, any}` 且该版本已实现的 category
3. 去重：每名干员取最高生产力技能（同精英阶段）
4. 枚举：`itertools.combinations(operators, 3)` 遍历所有三人组
5. 计算：根据各类机制规则累加生产力
6. 排序输出 Top 20

**基建假设（常量）：**
```python
POWER_PLANTS = 3    # 发电站
TRADING_POSTS = 2   # 贸易站
DORMS = 4           # 宿舍
DORM_LEVEL = 5      # 每宿舍等级
TRAINING_ROOM_LEVEL = 3  # 训练室等级
```

## 4. 实现进度跟踪

每次实现一个分类后，更新 `制造站_重新分类.md` 中的实现状态列（⬜→✅）。

**规则：**
- 每个新版本在旧版求解器基础上**增量添加**一类机制
- 添加后立即运行，对比新旧 Top 20 的变化
- 如果新机制改变了第一名，记录下来

**赤金潜力最高的复杂技能：**

|------|------|------------|
| 再生能源 | 每个贸易站+20% | 2贸易站=+40% |

## 6. 注意事项

- **仓库容量不影响赤金生产力排名**，仅当实现"仓库→产能"时相关
- **心情消耗**暂不纳入排名（可通过排班轮换解决）
- **中间产物**链路太长（需控制中枢配合），建议最后实现
- **配合意识(#46)** 是乘法效果，需要先计算其他两人的基础产能，再乘以倍数
- **归零类技能**之间互斥（同一制造站只能有一人带归零技能）
